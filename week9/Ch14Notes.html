HTML5 APIs

HTML5

  • The HTML5 specification is separated into modules that allow different features to be developed at different paces then implemented without having to wait for other features to be completed
  • when a previously unforeseen development occurs, a new module can be created to cater for it

The data- Attribute

  • The data- attribute is a way of embedding data in a web page using custom attributes that are ignored by the browser
  • private to a page
  • not intended to be used by an external service
  • sole purpose is to be used by a JavaScript program
  • provide a convenient way of adding data directly into the HTML markup, enabling a richer user experience

HTML5 APIs

  • help to gain access to hardware, such as cameras, batteries, geolocation, and the graphics card
  • best practice to use feature detection before using any of the API methods

HTML5 Web Storage

  • Web Storage API provides a key-value store on the client’s computer that is similar to using cookies but has fewer restrictions, more storage capacity, and is easier to use
  • Different from cookies
    • Information stored isnotshared with the server on every request
    • Information is available in multiple windows of the browser (if the domain is the same)
    • Storage capacity limit is much larger than the 4KB limit for cookie
    • data stored does not automatically expire as it does with cookies

Geolocation

  • Geolocation API is used to obtain the geographical position of the device
  • info can be used to filter data based on the user's location or speed and direction of travel
  • permission must be given first by user

Web Workers

  • Web workers allow processes to be run in the background, adding support for concurrency in JavaScript
  • Uses the Worker() constructor function
  • Web workers use the concept of messages to communicate back and forth between the main script and worker script

Shared Web Workers

  • can create shared web workers that allow lots of different scripts on the same domain to access the same worker object
  • allow computationally complex operations to be performed in a separate thread
  • an application will not freeze or hang

Service Workers

  • Service Worker API allows a worker script to run in the background with the added benefit of being able to intercept network requests
  • allows you to take alternative action if the network is offline
  • require a secure network to run on HTTPS to avoid any malicious code hijacking network requests

Websockets

  • a new protocol that allows two-way communication with a server – also known as push messaging
  • a connection is kept open and responses are ‘pushed’ to the client as soon as they are received
  • The close event occurs when the connection is closed. Done using the close() method
  • Error event is fired when any sort of error occurs with the connection.
  • information about the error can be accessed in the data property of the event object

Notifications

  • The Notification API allows you to show messages using the system's notifications
  • usually a popup in the corner of the screen
  • will still be displayed even if the web page that calls them isn't the current tab.
  • Depending on browser and operating system, some notifications close automatically after short period of time and some stay on the screen until user clicks on them

Multimedia

  • HTML5 introduced the audio and video tags used to insert audio and video clips into a web page
  • introduced a Media API for controlling the playback of the clips using JavaScript
  • audio and video elements bring native support for multimedia into the browser
  • API gives developers full control of the playback of audio tracks and video clips

Other APIs

  • comprehensive list of HTML5 APIs can be found at theMozilla Developer Network
  • security and privacy considerations to keep in mind when considering some HTML5 APIs

Drawing shapes with canvas

  • The canvas element was introduced to allow graphics to be drawn onto a web page in real time using JavaScript
  • A canvas element is a rectangular element on the web page
  • can now be accessed in a JavaScript program using the document.getElementById() method

Shims and Polyfills

  • can’t always guarantee that users will have the most up-to-date browser
  • libraries of code that allow you to use the APIs as usual. T fill in the necessary code not provided natively by the user’s browser
  • terms shim and polyfill are often used interchangeably
  • shim is a piece of code that adds some missing functionality to a browser
  • polyfill is a shim that achieves the same functionality while also using the API commands that would be used if feature was supported natively
  • don’t need to write additional code to deal with different levels of support
  • once users update their browsers you can remove the polyfill code without having to update any actual JavaScript code
