Notes for Chapter 15 (Modular JavaScript)

Modular JavaScript

  • self-contained code
  • code should have a single purpose
  • group functions with distinct functionality
  • makes it easy to swap out modules
  • can have a public API and keep other implmentation code hidden

Modules

  • all code is always in strict mode
  • has it's own global scope
  • 'this' in the top leve is 'undefined'
  • can't use HTML-style comments

Functions

  • functions can be exported from a module
  • when importing a lot of functions and values can use the wildcard symbol *
  • can be selective about which values or functions are imported

Default Exports

  • refers to a single variable, function, or class in a module
  • can not use more than one default export

Node.js Modules

  • uses a different notation
  • Common JS module pattern
