Transforms

  • CSS3 transform property lets you lets you translate, rotate, scale, and/or skew any element on the page
  • Can manipulate an element’s appearance usingtransform functions
  • The order of transform functions does matter

Translation

  • Translation functions allow you to move elements left, right, up, or down
  • Similar to position: relative;
  • Translated element can only be moved relative to its current position

Scaling

  • The scale(x,y) function scales an element by the defined factors horizontally then vertically
  • When one value is provided will be used for both the x and y values
  • Scaled element will grow outwards from or shrink inwards towards its center
  • Scaling, like translation, has no impact on the document flow

Rotation

  • The rotate() function rotates an element around the point of origin by a specified angle value.
  • Angles are declared in degrees

Skew

  • Specifies a skew along the x and y axes

Changing the Origin of the Transform

  • transform-origin property controls the origin from which transforms are applied.
  • Browser support for the transform-origin property is the same as for transform

Transitions

  • Transitions allow the values of CSS properties to change over time
  • Declared along with the regular styles on an element
  • Transition is declared in the default or originating state
  • transition-property defines the CSS properties of the element that should be transitioned
  • Must include a pre-state and a post-state.
  • Exception to this “if there is a valid midpoint, it can be transitioned” rule is visibility

The transition-duration Property

  • Sets how long the transition will take
  • 200ms is generally considered the optimum time for a transition

The transition-timing-function Property

  • Lets you control the pace of the transition in even more granular detail.
  • Most common timing functions include the key terms ease , linear , ease-in , ease-out , or ease-in-out
  • Can also describe timing function more precisely by defining own cubic-bezier function
  • Can divide the transition over equidistant steps

The transition-delay Property

  • Possible to introduce a delay before the transition begins
  • A negative time delay that’s less than the duration of the entire transition will cause it to start immediately

The transition Shorthand Property

  • Shorthand for the four transition properties
  • can be in any order
  • if a delay is included must also include a duration and duration must precede the delay

Multiple Transitions

  • allow for multiple transitions in one call
  • If want to change both properties at the same rate and delay can include both property names

Animations

  • allow you to control each step of an animation via keyframes
  • allow you to add any number of keyframes

Keyframes

  • To create animation, the @keyframes rule for IE10+ and FF16+. Include @-webkit-keyframes for all WebKit implementations followed by a choice of name, which is the identifier for the animation. Then, can specify keyframes
  • Instead of a traditional selector you use a percentage value, or a comma-separated list of percentage values
  • two keyterms― from and to ―which evaluate to 0% and 100% respectively
  • Inside each keyframe include the properties you want to animate along with animated values
  • An element must have at minimum an animation name for there to be an animation, and must also have a duration declared for the animation to be perceptible

Animation Properties

  • This property is used to attach an animation to an element
  • no quotes around the animation name in either the property value or the @keyframes at-rule
  • animation-duration defines the length of time (in seconds or milliseconds) an animation takes to complete one iteration
  • animation-name is the only required animation property to create an animation

animation-timing-function

  • determines how the animation will progress over its duration
  • options are the same as for transition-timing-function

animation-iteration-count

  • lets you define how many times the animation will play through
  • value is generally an integer
  • If omitted, it will default to 1

animation-direction

  • changes the behavior of the default or normal value for animation-direction
  • When animations are played in reverse, timing functions are also reversed

animation-delay

  • used to define how many milliseconds or seconds to wait before the browser begins the animation
  • The animation-fill-mode property defines what happens before the first animation iteration begins and after the last animation iteration concludes
  • By default, an animation has no effect on property values outside of when it’s iterating
  • available values are none , forwards , backwards , or both

The Shorthand animation Property

  • The animation property takes as its value a space-separated list of values for the longhand
  • avoid accidentally using any animation property key term such as forwards , running , or alternate
