Canvas

  • draw anything we can imagine, all through JavaScript
  • gives the power to manipulate pixels in images and even video
  • first developed by Apple

Creating a canvas Element

  • first step is to add a canvas element to the page
  • If not set, the canvas element will default to a width of 300 and a height of 150

Drawing on the Canvas

  • All drawing on canvas happens via the Canvas JavaScript API

Getting the Context

  • context is the place where drawing is rendered
  • Currently, there’s only wide support for drawing to a two-dimensional context
  • We obtain our drawing context by calling the getContext method and passing it the string "2d"
  • The object that’s returned by getContext is an instance of CanvasRenderingContext2D

Filling Brush with Color

  • fill brush with the strokeStyle or fillStyle properties

The Canvas Coordinate System

  • the coordinate system in the canvas element is different from the Cartesian coordinate system
  • the top-left corner is (0,0)

Anonymous Functions

  • much like regular functions except they are without names
  • means that the anonymous function is being bound to that event

Drawing Other Shapes by Creating Paths

  • can draw any shape we can imagine
  • no built-in method for drawing circles or other shapes
  • To draw more interesting shapes, we must first lay out the path of the shape
  • Pathscreate a blueprint for your lines, arcs, and shapes, but paths are invisible until they are given a stroke

Radians Explained

  • a unit used to measure angles
  • the symbol π denotes a measurement in radians

Saving Canvas Drawings

  • can use the API’s toDataURL method to save a drawing

Manipulating Images

  • really no different from using an img element
  • can use the getImageData method from the Canvas API to manipulate the pixels of that image
  • getImageData will return an ImageData object which contains three properties: width, height, and data
  • data contains information about the pixels in the ImageData object in the form of an array
  • Each pixel on the canvas will have four values in the data array, which correspond to that pixel’s R, G, B, and A values

Security Errors with getImageData

  • files loaded with the file:// protocol are considered to come from different domains, which is deemed a security error
  • security issue Chrome and Firefox are attempting to prohibit is a user on one domain manipulating images on another domain

Accessibility Concerns

  • major downside of canvas in its current form is its lack of accessibility
  • canvas doesn’t create a DOM node, is not a text-based format, and is essentially invisible to tools such as screen readers

SVG

  • stands forScalable Vector Graphics
  • a specific file format that allows you to describe vector graphics using XML
  • vector images preserve their quality even as you blow them up or shrink them down

Drawing in SVG

  • The viewBox attribute defines the starting location, width, and height of the SVG image
  • instead of creating SVG images by hand, we can use an image editor to help

Canvas versus SVG

  • best one - depends on what you're doing
  • Both canvas and SVG allow you to draw custom shapes, paths, and fonts
  • downside of canvas is that it operates in immediate mode
  • canvas is much faster than SVG
  • what you draw to SVG is accessible via the DOM
  • SVG also has a more complete set of tools
