Client-side form validation

  • to ensure all required form controls are filled out, in the correct format
  • to ensure data submitted matches the requirements set forth in the various form controls
  • it is an initial check
  • by catching invalid data on the client-side the user can fix it right away
  • should not be considered an exhaustive security measure

What is form validation?

  • form that provides feedback when you don't enter your data in the format it is expecting
  • Validation done in the browser is called client-side validation
  • validation done on the server is called server-side validation
  • Three reasons to validate forms
    • We want to get the right data, in the right format
    • We want to protect our users' data
    • We want to protect ourselves

Different types of client-side validation

  • Built-in form validation uses HTML5 form validation features
  • JavaScript validation is coded using JavaScript

Using built-in form validation

  • One of the most significant features of HTML5 form controls is the ability to validate most user data without relying on JavaScript
  • done by using validation attributes on form elements

Validating forms using JavaScript

  • must use JavaScript if want to take control or to deal with legacy browsers that do not support HTML's built-in form validation

The Constraint Validation API

  • consists of a set of methods and properties available on the form element DOM interfaces

Validating forms without a built-in API

  • To validate, ask yourself a few questions:
    • What kind of validation should I perform?
    • What should I do if the form doesn't validate?
    • How can I help the user to correct invalid data?
