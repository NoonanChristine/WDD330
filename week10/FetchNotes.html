Using Fetch

  • The Fetch API provides a JavaScript interface for accessing and manipulating parts of the HTTP pipeline, such as requests and responses
  • also provides a global fetch() method
  • The Promise returned from fetch() won’t reject on HTTP error status even if the response is an HTTP 404 or 500
  • won’t send cross-origin cookies unless you set the credentials init option
  • The fetch() method can optionally accept a second parameter, an init object

Supplying Request Options

  • fetch() method can optionally accept a second parameter (an init object) to control settings

Sending a Request With Credentials Included

  • add credentials: 'include' to the init object you pass to the fetch() method
  • Browsers should not send credentials in preflight requests

Uploading JSON data

  • Use fetch() to POST JSON-encoded data

Uploading a File

  • Files can be uploaded using an HTML input type="file" input element, FormData() and fetch()

Uploading Multiple Files

  • Files can be uploaded using an HTML input type="file" multiple input element, FormData() and fetch()

Processing a text file line by line

  • The chunks that are read from a response are not broken at line boundaries and are Uint8Arrays

Checking that the fetch was successful

  • A fetch() promise will reject with a TypeError when a network error is encountered or CORS is misconfigured on the server-side
  • accurate check for a successful fetch() would include checking that the promise resolved and then checking that the Response.ok property has value of true

Supplying Your Own Request Object

  • can create a request object using the Request() constructor, and pass that in as a fetch() method argument
  • Request() accepts exactly the same parameters as the fetch() method

Headers

  • The Headers interface allows creation of own headers object via the Headers() constructor
  • headers object is a simple multi-map of names to values
  • All Headers methods throw a TypeError if a header name is used and is not a valid HTTP Header name

Guard

  • headers'objects have a guard property
  • affects which mutation operations are allowed on the headers object

Response Objects

  • Response instances are returned when fetch() promises are resolved
  • can also be created programmatically via JavaScript

Body

  • Both requests and responses may contain body data
  • an instance of ArrayBuffer, ArrayBufferView, Blob/File, string, URLSearchParams, or FormData
  • Request bodies can be set by passing body parameters

Feature Detection

  • Fetch API support can be detected by checking for the existence of Headers, Request, Response or fetch() on the Window or Worker scope

Polyfill

  • recreates the functionality for non-supporting browsers
